## 一、项目背景与核心定位

### 一句话定义
> "你今天练什么，我们就给你派什么活；每一次深蹲、每一次爬楼、每一次搬运，都有人为你的训练买单。"

### 与企业愿景的关系
Superhero APP 是「全民普适性价值创造」体系的**旗舰落地项目**：
- 将「全民终身学习」升级为「全民终身创造」
- 把「反人性的健身消耗」变成「有即时收益的正向反馈」
- 让普通人从「价值分配末端」走向「价值创造前端」
- 实现「用户得收益/健康 + 平台得佣金 + 社会解轻功/假勤需求」三方共赢

### 差异化（与美团/Keep等本质区别）

| 维度 | 美团跑腿 | Keep健身 | Superhero APP |
|------|----------|----------|---------------|
| 核心动机 | 赚钱→顺便走路 | 健康→消耗卡路里 | 训练计划→匹配任务 |
| 用户动机 | 赚钱优先 | 健康优先 | 健身优先，赚钱是奖励 |
| B端价值 | 快速交付 | 无B端能力 | 精准匹配"动作需求"，成本低30% |
| 训练属性 | 无 | 有，但无收益 | 训练即任务，任务即训练 |

## 二、战略决策框架（科学决策）

### 核心假设（待验证）
1. 营口社区商户有碎片化轻体力劳务需求（配送/搬运/巡检）
2. 营口居民愿意为「健身+赚钱」的模式参与
3. 日均任务量能支撑供给侧稳态

### 关键约束
- 投入上限：单社区试点 10-15万元
- 回本周期：要求 6-12个月内跑通盈利模型
- 合规底线：不碰食品/医疗等强监管领域（初期）
- 落地优先：营口本地先行

### 决策树（优先验证）

```
阶段一：调研 → 判断：
  营口社区密度是否达标？→ 是→继续 / 否→调整策略
  商户轻劳务需求真实存在？→ 是→继续 / 否→换场景（乡镇/景区）
  用户参与意愿 > 30%？→ 是→继续 / 否→降补贴或换模式

阶段二：MVP（单社区）→ 判断：
  日均任务匹配率 > 60%？→ 是→复制 / 否→优化算法
  用户周留存 > 40%？→ 是→复制 / 否→调整激励
  单社区月GMV > 3万元？→ 是→复制 / 否→控成本

阶段三：单城市 × N社区 → 判断：
  跨社区可复制？→ 是→标准化扩张 / 否→本地化调整
  单城市月纯利 > 10万元？→ 是→规模化 / 否→优化模型
```

### Phase 1: MVP 设计与开发（2026-05-26 — 2026-06-25，约1个月）
- 基于调研结论，确定 MVP 功能边界
- "先做小程序，不做APP"（降低开发成本+推广门槛）
- 核心功能：用户注册→选择训练计划→匹配附近任务→完成获得收益
- B端管理台：一键发布任务模板
- 游戏化系统：等级/积分/成就
- **预计消耗token**: 约15000-25000 tokens（代码生成+调试）

### Phase 2: 营口单社区试点（2026-06-26 — 2026-08-25，约2个月）
- 选择营口1-2个大型成熟社区（>10000户）
- 对接20-30家社区商户 + 1家物业 + 1个居委会
- 招募200-300个种子用户
- 验证：任务匹配率、用户留存、B端复购率
- **预计消耗token**: 约10000-15000 tokens（运营迭代）

### Phase 3: 营口多社区扩张（2026-08-26 — 2026-10-25）
- 基于试点数据优化产品和运营
- 扩展至5-10个社区，覆盖10万+人口
- 实现单社区盈亏平衡
- **预计消耗token**: 约8000-12000 tokens

### Phase 4: 单城市盈利 & 模式标准化（2026-10-26 — 2026-12-31）
- 覆盖营口主要城区
- 建立标准化运营手册
- 目标：月纯利 > 10万元
- 为跨城市复制做准备
- **预计消耗token**: 约5000-8000 tokens

## 六、风险和应对策略

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 营口市场太小，无法支撑模型 | 中 | 高 | 转乡镇/景区场景，或降低期望日均订单量 |
| 商户不愿意用 | 中 | 高 | 前期免费试用，先用巡检/社区公益任务练手 |
| 用户留存率低 | 中 | 中 | 强化游戏化设计，引入等级/成就/团队PK |
| 保险合作谈不下来 | 低 | 中 | 先自保（平台兜底小额意外），用户量大了再谈 |
| 出现跟风竞争者 | 低 | 中 | 深耕营口本地护城河，先发优势+社区关系壁垒 |

## 七、决策记录

### 已决策（2026-05-13）

1. **MVP 边界** → 全支持。配送、搬运、巡检等所有轻体力任务类型全部纳入 MVP。
2. **盈利策略** → 
   - 前期可适当赔本，不急于盈利
   - 非金额类奖励（权益/积分/公益/碳积分等）**不抽佣金**
   - 金额类奖励 **抽 10%**
   - 短期靠「爱心」：公益价值、社会认可带动口碑
   - 中期靠「广告」：商户及品牌广告收入
   - 长期靠「保险佣金」：浮动阶梯佣金 + 风控服务费
3. **冷启动** → 先调研启动难点在哪，再决定具体路线
4. **品牌命名** → 需要改名，不为本土化，而是为全国铺开的品牌化做准备
5. **资源分配** → 
   - 反诈小游戏：**搁置**
   - 反诈论文：**搁置**
   - 公众号文章：**保持每日一篇不中断**
   - 其他未列项目：优先级随Superhero推进适当下降

## 八、品牌命名思路

> Superhero「超能侠」暂用名定位明确但过于直白，需要为全国铺开储备一个更有层次感的名字。以下是按不同品牌调性分组的思路：

### 调性A：理念驱动型（重「利他+价值感」）
| 备选名 | 思路 | 适配场景 |
|--------|------|---------|
| **值道** | 每一次行动都有价值，「值」= 价值/值得，「道」= 道理/道路 | 品牌Slogan：「每一步都值得」|
| **众行** | 众人同行，聚沙成塔 | 强调社区互助、集体感 |
| **微力** | 微小的力量汇聚成大的价值 | 适合下沉市场，亲切不装 |
| **必达** | 使命必达，同时又隐含「必需达到健身目标」的双关 | 简洁有力，记忆成本低 |

### 调性B：游戏化驱动型（重「有趣+上瘾」）
| 备选名 | 思路 | 适配场景 |
|--------|------|---------|
| **练了** | 口语化、高识别，「练了吗？」形成社交传播 | 适合年轻人、校园市场 |
| **成事** | 「成事不足」→「成事健身」的反转梗 | 有趣，容易形成社区黑话 |
| **活易** | 动一动就赚钱，把「生活不易」变成「生活易」 | 正反馈轻快，记忆点强 |
| **到位** | 任务到位、健身到位、钱到位 | 双关，简洁 |

### 调性C：功能驱动型（重「直接+信任感」）
| 备选名 | 思路 | 适配场景 |
|--------|------|---------|
| **快跑** | 既是跑步健身，又是快速跑腿 | 简单直接，零认知成本 |
| **代步** | 代替步行=赚钱，一步换一步 | 适合偏跑腿/配送的场景 |
| **顺动** | 顺路运动，顺手赚钱 | 形象生动 |
| **近助** | 附近的人互相帮助 | 强调社区属性 |

### 调性D：品牌资产型（重「可讲故事+IP延伸」）
| 备选名 | 思路 | 适配场景 |
|--------|------|---------|
| **途光** | 在路上发光，「途」=路途/征途，「光」=荣誉/希望 | 适合做IP和周边 |
| **拾金** | 「走路拾金」的谐音记忆点，每一步都捡到宝 | 记忆门槛最低 |
| ⭐ **星火** ✅ | 星星之火可以燎原，也寓意每个人的微小贡献 | **已选中**，与「灵星共成」天然联动 |
| **壹步** | 千里之行始于足下，也暗示「一起」的谐音 | 简洁好记 |

### 命名评估框架
> - **记忆成本**：3秒内能不能记住
> - **品类联想**：能不能联想到「健身+任务」
> - **传播性**：适不适合社交传播（有没有梗/谐音）
> - **抗老化**：5年后还合不合适
> - **商标可用性**：是否被注册（需查商标局）
